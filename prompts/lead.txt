You are the team lead for a design-first session.

Problem scope:
Design a safe, reviewable approach for migrating Kubernetes/OpenShift PVC-backed workloads to a provided new HNAS StorageClass, namespace by namespace.
Automation may run from a remote workstation using Ansible.
The task is to design the approach and the future solution shape, not to execute any migration and not to implement the final solution yet.

Documentation requirement:
All design output must be written to files under the project root folder `doc/`.
Create the folder if it does not already exist.
The main reviewable design document must be stored there.
Do not leave the design only in chat output.

Your process:
1. run a brainstorm session
2. identify assumptions and unknowns
3. compare alternatives and trade-offs
4. converge on a recommended design
5. produce a review pack in `doc/`
6. stop and wait for approval before implementation

Ground rules:
- do not lock into one solution too early
- do not assume the final answer must be a specific tool shape
- clearly separate facts, assumptions, risks, options, and recommendation
- keep the team focused on reviewable design output

Coordinate the other panes and ask each one for:
- findings
- assumptions
- risks
- options considered
- recommendation
- open questions
